# Personas and Roles

## Parent

### Goals

- contribute money or time
- understand what is happening
- avoid unnecessary meetings
- see proof of spending
- submit feedback without friction

### Pain points

- too many messages in chat groups
- unclear payment requests
- social pressure
- lack of time
- language barriers
- distrust in informal collections

### Permissions

- manage own profile
- view public reports
- view class-related tasks
- claim tasks
- submit feedback
- create ideas if enabled
- register for events

## Parent Committee Representative

### Goals

- coordinate parents
- keep transparent records
- collect contributions legally
- publish reports
- prioritize school needs

### Permissions

- manage tasks
- manage expenses
- confirm payments manually
- moderate feedback
- publish news
- export reports

## Teacher

### Goals

- communicate class needs
- submit tasks
- review class-related feedback
- avoid direct handling of money

### Permissions

- create tasks for own class
- view tasks for own class
- respond to feedback assigned to them
- create events

## School Staff

### Goals

- publish operational needs
- coordinate school-wide tasks
- receive structured feedback
- keep separation from committee finances

### Permissions

- create school-wide tasks
- manage events
- view relevant feedback
- cannot manage contributions unless explicitly assigned

## Administrator

### Goals

- configure the system
- manage access
- support users
- investigate incidents

### Permissions

- full system management
- role assignment
- audit log access
- exports
- configuration

## Child

### MVP status

Designed for future use. Not enabled in MVP by default.

### Future permissions

- submit idea
- like/vote on ideas
- see approved ideas
- no private messaging
- no unrestricted comments
- no public personal profile

# Child Safety and Moderation

## MVP recommendation

Do not enable direct child accounts in MVP.

Instead:

- parent submits child idea on behalf of child
- committee moderates
- idea can be published after approval

## Future child zone

If enabled, strict limits:

- no private messaging
- no free public profile
- no public full names
- no unmoderated comments
- no personal photos
- no location sharing
- no direct contact between adults and children

## Idea moderation

Statuses:

- submitted
- approved
- voting
- funding
- converted_to_task
- realized
- rejected

## Comment moderation

MVP can avoid comments on ideas.

If comments are enabled:

- all child-facing comments moderated
- report button
- block abusive words
- audit moderation actions

## Anti-bullying controls

- no downvotes
- no negative reactions
- no ranking children
- no public rejection reason targeting child
- no direct peer messaging

## Parent consent

Child-related features require parent consent.

Consent must cover:

- idea submission
- display of idea title/description
- class context if shown

## Safe display

Public idea card should show:

- idea title
- class optional
- status
- votes

Avoid:

- child full name
- child photo
- personal story

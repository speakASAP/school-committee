## Purpose

A context package identifies the minimum documents an implementation agent must read for a task.

## Naming

Use `CP-<TASK-ID>.md` under `docs/13_context_packages/`.

## Required sections

Each context package includes Target task, Upstream traceability, Included documents, Excluded documents, Constraints, Agent prompt, and Validation instructions.
